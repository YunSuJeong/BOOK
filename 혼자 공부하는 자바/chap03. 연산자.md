# 연산자

## 03-1. 연산자와 연산식
`연산자(operator) : 연산에 사용되는 표시나 기호`  
`피연산자(operand) : 연산자와 함께 연산되는 데이터`  
`연산식(expression) : 연산자와 피연산자를 이용하여 연산의 과정을 기술한 것`

### ▶︎ 연산자 종류
- 연산 결과의 타입은 연산자별로 다르다.  
- 연산식은 반드시 하나의 값을 산출하며, 값 대신에 연산식을 사용할 수 있다.

| 연산자 종류 | 연산자 | 피연산자 수 | 산출값 | 기능 |
|---|---|---|---|---|
| 산술 | +, -, *, /, % | 이항 | 숫자 | 사칙연산 및 나머지 계산 |
| 부호 | +, - | 단항 | 숫자 | 음수와 양수의 부호 |
| 문자열 | + | 이항 | 문자열 | 두 문자열을 연결 |
| 대입 | =, +=, -=, *=, /=, %= | 이항 | 숫자 | 우변의 값을 좌변의 변수에 대입 |
| 증감 | ++, -- | 단항 | 숫자 | 1만큼 증가/감소 |
| 비교 | ==, !=, >, <, >=, <=, instanceof | 이항 | boolean | 값의 비교 |
| 논리 | !, &, |, &&, || | 단항, 이항 | boolean | 논리 부정, 논리곱, 논리합 |
| 조건 | (조건식) ? A : B | 삼항 | 다양 | 조건식에 따라 A 또는 B 중 하나를 선택 |

### ▶︎ 연산의 방향과 우선순위
- **대부분의 연산자는 왼쪽에서 오른쪽 방향**으로 연산 수행한다.  
- 예외적으로 오른쪽에서 왼쪽방향으로 연산을 수행하는 연산자도 있다. (아래 표 참조)  
- 괄호 ()를 이용하여 우선순위를 정할 수 있다.

| 연산자 | 연산 방향 | 우선순위(아래로 갈수록 낮다) |
|---|---|---|
| 증감(++,--), 부호(+,-), 논리(!) | ← | ▲ |
| 산술(*, /, %) | → ||
| 산술(+, -) | → ||
| 비교(<, >, <=, >=, instanceof) | → ||
| 비교(==, !=) | → ||
| 논리(&) | → ||
| 논리(^) | → ||
| 논리(\|) | → ||
| 논리(&&) | → ||
| 논리(\|\) | → ||
| 조건(?:) | → ||
| 대입(=, +=, -=, *=, /=, %=) | ← | ▼ |



## 03-2. 연산자의 종류
> 연산자는 필요로 하는 **피연산자의 수에 따라 단항, 이항, 삼항 연산자**로 구분된다.

### ▶︎ 단항 연산자
> 피연산자가 단 하나뿐인 연산자  
> 부호, 증감, 논리 부정 연산자가 있다.

#### 📌 부호 연산자(+, -)
- 정수 및 실수 리터럴 앞에 붙여 양수 및 음수를 표시
- boolean, char 타입을 제외한 나머지 기본 타입에 사용할 수 있다
- **부호 연산의 결과는 int타입이므로 int 타입 변수에 저장해야한다.**

```java
byte b = 100;

byte result = -b;      // 컴파일 에러
int resut = -b;
```

#### 📌 증감 연산자(++, --)
- 변수의 값을 1 증가시키거나, 감소시키는 연산자
- boolean타입을 제외한 모든 기본 타입 피연산자에 사용할 수 있다
- **증감 연산자의 위치에 따라 연산식의 결과가 다르게 나올 수 있다**
  - 증감 연산자만 사용된 연산식에서는 위치 상관없이 동일한 결과를 얻음
  - **다른 연산자와 함께 사용될 때 주의**
- ++i와 i = i + 1 연산 속도는 차이가 없다.

| 연산식 | 설명 |
|---|---|
| (++, --)피연산자 | 다른 연산을 수행하기 전에 피연산자 값을 1 증가/감소 |
| 피연산자(++, --) | 다른 연산을 수행한 후에 피연산자의 값을 1 증가/감소 |

```java
int x = 1;
int y = 1;
int result1 = ++x + 10;    // 2(x가 먼저 증가) + 10 = 12
int result2 = y++ + 10;    // 1 + 10 = 11 연산 후, y감 증가
```

#### 📌 논리 부정 연산자(!)
- true ↔︎ false 변경하는 연산자
- boolean 타입에만 사용할 수 있다
- 조건문과 제어문에서 조건식의 값을 부정하도록 하여 실행 흐름을 제어할 때 주로 사용

### ▶︎ 이항 연산자
> 피연산자가 2개인 연산자  
> 산술, 문자열 결합, 비교, 논리, 대입 연산자 등이 있다.

#### 📌 산술 연산자(+, -, *, /, %)
- 사칙연산자
- 덧셈(+), 뺄셈(-), 곱셈(*), 나눗셈(/), 나머지(%)
- boolean타입을 제외한 모든 기본타입에 사용할 수 있다
- **피연산자들의 타입이 동일하지 않을 경우 다음 규칙을 사용해 연산을 수행**
  - 피연산자들이 byte, short, char 타입인 경우 → 모두 int 타입으로 변환 후 연산
  - 피연산자들이 모두 정수타입이고 long타입이 포함되어 있는 경우 → 모두 long 타입으로 변환 후 연산
  - 피연산자 중 실수 타입(float, double)이 있는 경우 → 허용 범위가 큰 실수 타입으로 변환된 후 연산을 수행

```java
byte byte1 = 1;
byte byte2 = 1;
byte byte3 = byte1 + byte2;      // 컴파일 에러, 연산 결과는 int형에 저장해야함
```

#### 📌 문자열 결합 연산자(+)
- 문자열을 서로 결합하는 연산자
- 피연산자 중 한쪽이 문자열이면 + 연산자는 문자열 결합 연산자로 사용된다

#### 📌 비교 연산자(<, <=, >, >=, ==, !=)
- 피연산자의 대소 또는 동등을 비교해서 true/false를 산출
- 대소 연산자(<, <=, >, >=)는 boolean을 제외한 기본 타입에서 사용할 수 있다
- 동등 연산자(==, !=)는 모든 기본 타입에 사용할 수 있다
- **연산을 수행하기 전에 피연산자의 타입을 일치시킨다**
  - char와 int형의 비교면 int타입으로 변환
  - 정수와 실수타입 비교면 실수 타입으로 변환
  - **예외적으로 float와 double형은 비교할 수 없다. 무조건 false 반환됨**
    - 실수의 저장 방식인 부동 소수점 방식이 소수를 정확히 표현 할 수 없어 근사값으로 표현되기 때문

#### 📌 논리 연산자(&&, ||, &, |, ^, !)
- 논리곱(&&), 논리합(||), 배타적 논리합(^), 논리 부정(!) 연산을 수행
- 피연산자는 boolean 타입만 사용할 수 있다

- &&와 &는 연산 과정에 차이가 있다 (||와 |도 마찬가지!)
  - && : 앞의 피연산자가 false면 뒤 피연산자를 평가하지 않고 바로 false 산출 결과를 낸다
  - & : 두 피연산자를 모두 평가하여 산출 결과를 낸다

#### 📌 대입 연산자(=, +=, -=, *=, /=, %=)
- 오른쪽 피연산자의 값을 왼쪽 피연산자인 변수에 저장한다
  - 오른쪽 피연산자에는 리터럴 및 변수, 다른 연산식이 올 수 있다
- 단순 대입 연산자와 복합 대입 연산자로 나뉜다
  - 단순 대입 연산자 : 오른쪽 피연산자의 값을 변수에 저장
  - 복합 대입 연산자 : 정해진 연산을 수행한 후 결과를 변수에 저장
- **모든 연산자들 중에 가장 낮은 연산 순위기 때문에 제일 마지막에 수행된다**
- 연산 진행 방향 : 오른쪽에서 왼쪽

### ▶︎ 삼항 연산자 (?:)
- 3개의 피연산자를 필요로 하는 연산자이다.
- ? 앞의 조건식에 따라 : 앞뒤의 피연산자가 선택된다.
- 조건 연산식이라고도 부른다.
